## New Parsers — File Format Support

### CSVs (general files with mixed formatting)
- [x] **Arbitrary headers and empty columns between data** — `importCSV` now handles:
    - **Empty separator columns**: fully-empty columns (e.g. blank columns between Excel data groups)
      are silently removed by the existing numericFrac filter; blank column headers are auto-renamed to `Col{N}`
    - **Dedicated units rows**: a second non-numeric row immediately before the first data row (e.g.
      `"(s)","(°C)","emu"`) is detected as a units row; values are used for channel units and override
      any inline `"Label (unit)"` extraction from the header
    - **Pre-header metadata**: all non-comment text rows before the column header row are captured and
      stored in `data.metadata.parserSpecific.headerMetadata` (instrument name, date, operator, etc.)
    - **Column alignment**: header is padded/trimmed to match actual data row width

### XRD (XML-based, no example files needed)

- [x] **`importXRDML.m`** — PANalytical / Malvern Empyrean `.xrdml` format. Well-documented
  XML schema; parse with `readstruct` or `xmlread`. Extract 2θ array, intensity counts,
  step size, wavelength, and scan metadata. **Validated with real La₂NiO₄ file; smoke test PASS.**

- [/] **`importBruker.m`** — Bruker D8/D2 `.brml` (ZIP archive containing XML) and legacy
  `.raw` binary (v3 magic `"RAW1.01"`, v4 magic `"RAW "`). Unzip `.brml` with `unzip`,
  then XML parse; binary `.raw` requires manual byte-level reading similar to
  `importRigaku_raw.m`. Single parser dispatches on extension/magic.

- [x] **handle panalytical multidimensional x-ray data** datasets from pixel modern panalytical
    3d detector puts data out in a area detector which the current parser cannot handle well. add capability
